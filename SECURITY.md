# Security policy

보안 취약점은 공개 issue보다 GitHub 저장소의 **Private vulnerability reporting** 기능으로
제보해 주십시오. 재현 절차, 영향 범위, 사용한 파일/요청, 확인한 버전을 포함해 주시면 됩니다.

현재 `0.x` 개발 단계에서는 최신 main과 최신 release만 보안 수정 대상입니다. secret,
presigned URL, 실제 사용자 미디어는 제보 본문에 첨부하지 마십시오.

제품 위협 모델과 기본 통제는 [docs/SECURITY.md](docs/SECURITY.md)를 따릅니다.
