# G7 실제 저장소 브라우저 E2E 증거

- 기준 G7: `e64381ddb5ba02caed60933427fbb86ef72ef94e`
- G7MediaBooster module: `0.3.0`
- 격리 환경: 별도 worktree, MySQL 8.4, MinIO, PHP 8.5.3, 실제 Rust API·worker·sandbox
- 적용 patch: `0001` → `0002` → `0003` → `0004`
- 원본 G7 작업 트리: 미수정

## 통과 결과

- native PHP uploader가 제거되고 G7MediaBooster direct uploader만 렌더링: PASS
- 402-byte JPEG single PUT와 9,796,467-byte JPEG 2-part multipart를 한 번에 전송: PASS
- 두 파일 모두 Rust 격리 검사·EXIF 제거·master/thumbnail 생성 후 `Ready`: PASS
- G7 native attachment 2개 materialize, `attachments_count=2`, 파일명·크기 표시: PASS
- 게시글 PUT 수정 후 attachment 2개 유지: PASS
- 두 thumbnail 모두 G7 권한 bridge `302`, private MinIO `200 image/jpeg`: PASS
- thumbnail 응답 크기: 505 bytes, 240,658 bytes

## E2E에서 발견해 고정한 결함

1. 조건부 basic/gallery/card 분기에 같은 layout ID가 반복될 때 첫 target만 overlay되던 문제를
   `0003`으로 고쳤습니다. 모든 일치 target에 component와 props를 적용하는 회귀 테스트를 추가했습니다.
2. `attachment_ids` bulk link가 모델 hook을 우회해 `attachments_count=0`이던 문제를 `0004`로
   고쳤습니다. 같은 transaction에서 count를 재계산하는 통합 테스트를 추가했습니다.
3. Rust `OffsetDateTime`이 배열 JSON으로 직렬화돼 PHP RFC3339 validator가 private delivery를
   `502`로 거부하던 문제를 RFC3339 string 계약으로 고쳤습니다. batch·part·delivery 하네스가
   모든 `expires_at` 필드를 문자열로 검사합니다.

## 공식 범위 밖

이 결과는 격리 MinIO profile과 현재 소유자 create/update 경로의 증거입니다. upstream patch 정식
반영, 권한 차단의 실브라우저 매트릭스, attachment 삭제 command→실 provider 객체 삭제와
실 R2/Lightsail profile은 별도 게이트입니다. 서버 권한·보존 DB gate는
[G7 보안·보존 gate](G7_SECURITY_RETENTION_GATE_20260716.md)에서 통과했습니다.
