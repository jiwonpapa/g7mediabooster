# MOV/H.264 runtime·worker 종단 증거

- 실행일: 2026-07-16
- 범위: pinned MinIO, 실제 Rust API/worker/sandbox, G5/G7 PHP·TypeScript adapter
- 외부 계정: 사용하지 않음

## Runtime capability

`g7mb-sandbox capabilities`는 내장 MP4/H.264를 실제로 검사한 뒤 MOV로 remux하여 두
container 모두 FFprobe 검사와 FFmpeg JPEG poster 추출을 수행합니다. 검증된 응답은 다음
필드를 포함하며 MP4 또는 MOV가 빠지면 API startup이 fail-closed합니다.

```json
"video_inputs":["mov","mp4"]
```

## 실제 worker 종단

실행 명령:

```bash
cargo xtask full-stack-smoke
```

결과:

```text
full-stack-smoke PASS single_put=1 multipart_parts=2 ready=3 derivatives=6 mov_h264=1 exif_removed=1
```

하네스가 FFmpeg로 실제 MOV/H.264를 생성한 뒤 `video/quicktime` intent로 multipart 직접
업로드합니다. worker는 QuickTime으로 판정하고 입력과 byte-identical한 `.mov` master와
decode 가능한 JPEG poster를 같은 preset으로 발행합니다. API private delivery가 master의
`video/quicktime`과 poster의 `image/jpeg`를 보존하는 것도 확인합니다.

## Adapter gate

- G7: PHP 57 tests/153 assertions, TypeScript 21 tests, typecheck/build PASS
- G5: PHP 17 tests/31 assertions, TypeScript 5 tests, typecheck/build PASS

G5/G7 모두 `.mov`와 `video/quicktime`을 업로드 intent, Ready materialization, private
delivery에서 허용합니다. detected container와 master MIME이 다르면 fail-closed하며 WebM은
release fixture 전 사용자 업로더에서 계속 거부합니다.

## 범위 경계

MOV의 OpenH264 폴백은 지원하지 않습니다. MOV는 FFmpeg 주 경로가 성공할 때만 처리하며,
FFmpeg 실행 불가 폴백은 기존 MP4/H.264 한정입니다. 영상 transcoding과 metadata 제거도
v1 공식 범위가 아닙니다.
